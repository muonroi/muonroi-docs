# UI Engine Architecture

The UI engine keeps policy and permission decisions on the backend and renders them on the frontend through metadata.

## Design principles

- backend-first manifests
- adapter-driven rendering
- rule-authoring widgets as reusable web components
- React and framework adapters layered on top of custom elements

## Current packages

- OSS: core, React, Angular, PrimeNG
- Commercial: rule components, SignalR helpers, sync tooling
